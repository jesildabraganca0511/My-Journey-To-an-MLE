IAM Section – Summary

Users: Represent individual people who can log in to the AWS Management Console using a username and password.

Groups: Collections of users that allow permissions to be managed for multiple users at once.

Policies: JSON documents that define the permissions granted to users, groups, or roles.

Roles: Identities that AWS services (like EC2) can assume to gain temporary permissions.

Security: Includes features such as Multi-Factor Authentication (MFA) and password policies to enhance account protection.

AWS CLI: A command-line tool used to interact with and manage AWS services.

AWS SDK: Tools that allow developers to interact with AWS services using programming languages.

Access Keys: Credentials used to access AWS services programmatically through the CLI or SDK.

Audit: Tools like IAM Credential Reports and IAM Access Advisor help monitor and review user permissions and security practices.
